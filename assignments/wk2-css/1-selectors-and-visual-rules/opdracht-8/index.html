Styling die je toepast wordt toegepast in een bepaalde volgorde. Het kan voorkomen dat je styling toepast en vervolgens niet het gewenste resultaat ziet. Cascade en Inheritance zijn belangrijke concepten om te begrijpen hoe styling werkt. Lees hier in de officiële documentatie meer over cascade en inheritance. Zoek ook andere referenties om het concept te begrijpen.

Leg in eigen woorden uit wat met cascade en inheritance wordt bedoeld. Maak voorbeelden om je antwoord uit te leggen.

“Stylesheets cascade”, dit betekent dat de volgorde van je CSS regels bepalend zijn. Wanneer er twee regels van toepassing zijn op een element met dezelfde uitwerking, dan wordt de regel die je als laatst in je CSS hebt toegevoegd gebruikt.

Wanneer ik onderstaande class eerst de kleur groen toeken, en daarna de kleur blauw, zal deze blauw worden.

.example1{color:green} .example1{color:darkblue}

Inheritance bepaalt wat er gebeurt wanneer er geen waarde wordt toegekend aan een property van een element. Er zijn twee soorten properties.

  • inherited properties, which by default are set to the computed value of the parent element
  • non-inherited properties, which by default are set to initial value of the property

Je kunt in het overzicht van de CSS properties opzoeken of een property iets by default overneemt, of niet.

Some CSS property values set on parent elements are inherited by their child elements, and some aren't. For example, if you set a color and font-family on an element, every element inside it will also be styled with that color and font, unless you've applied different color and font values directly to them.

In het volgende voorbeeld heeft een div met de class example2 als kleur green meegekregen. Het p element dat hierin zit neemt deze kleur ook over.

.example2 {color:green}
