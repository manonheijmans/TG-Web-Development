a. Onderzoek de volgende DOM methodes en gebruik ze in een voorbeeld:

i. document.getElementsByClassName

class example

    <p class="select-by-class">class example</p>

    let classSelector = document.getElementsByClassName('select-class');
        
    console.log(classSelector)

Dit logt het volgende: HTMLCollection { 0: p.select-class, length: 1}


ii. document.getElementsByTagName

     <button>tag example</button>
 
     let tagSelector = document.getElementsByTagName('button');

     console.log(tagSelector)

Dit logt het volgende: HTMLCollection { 0: button, length: 1}


iii. document.querySelector

Deze p met class "select-query" wordt wel geselecteerd.

Deze p met class "select-query" wordt niet geselecteerd.

     <p class="select-first-query">Deze p met class "select-query" wordt wel geselecteerd.</p> 
     <p class="select-first-query">Deze p met class "select-query" wordt niet geselecteerd.</p> 

     let querySelectorExample = document.querySelector('.select-query');

     console.log(querySelectorExample)

Dit logt het volgende: <p class="select-query">


iv. document.querySelectorAll

In dit geval worden de twee p's hierboven, in tegenstelling tot met de document.querySelector allemaal in een statische (niet live) NodeList weergegeven.

     let querySelectorAllExample = document.querySelectorAll('.select-query');

     console.log(querySelectorAllExample)

Dit logt het volgende: NodeList [p.select-query, p.select-query]


v. element.innerHTML

  • Deze list wordt met element.innerHTML gelogd.
  • Deze list wordt met element.innerHTML gelogd.
      
    <ul id="innerHTMLSelector">

        <li>Deze list wordt met element.innerHTML gelogd.</li>

vi. element.innerText

Deze text wordt met element.innerText gelogd.

      
    <p id="innerTextSelector">Deze text wordt met element.innerText gelogd.</p>
   

    let innerTextExample = document.getElementById('innerTextSelector');

    console.log(innerTextSelector.innerText)

De log geeft het volgende: Deze text wordt met element.innerText gelogd.

b. Bij opdracht 4.1c heb je een input als parameter meegegeven. Roep die functie nu aan met een oninput ipv onchange, en console.log() de parameter.value ipv de parameter. Zie je het verschil in beide gevallen? 


        <input id="onInputExample" oninput=onInputFunction(onInputExample) placeholder="onInputExample">

        let onInputVar = document.getElementById('onInputExample')

        function onInputFunction(parameter) {
                console.log(parameter.value)
        }

        <input id="onChangeExample" onchange=onChangeFunction(onChangeExample) placeholder="onChangeExample">

        let onChangeVar = document.getElementById('onChangeExample')

        function onChangeFunction(parameter) {
                console.log(parameter.value)
        }

Bij console.log() geeft oninput een live weergave van wat er op dat moment ingevoerd wordt. Wanneer je 123 intikt, logt ie zowel 1, 12, 123. Bij onchange wordt er echter pas gelogd wanneer je de invoer doorgeeft. In dit geval door middel van enter.
